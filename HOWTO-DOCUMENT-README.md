# Howto Document
The purpose of a README file is to provide the reader with an easy-to-understand and well-structured overview of a topic.
It serves as a central starting point that provides access to further documentation and resources through direct links.

# Basic Guide

## Overall Structure
1. **Headline** </br> Start with a clear and informative title. This should immediately indicate what the document is about. Also possible is to use the folder's name as the headline.
2. **Introduction and Description** </br> Introduce the topic and state the 
   - Content 
   - Purpose
3. **Linking** </br> All files in the folder should be linked in the README file. They can either be presented as a bullet-point list or included in the text.
4. **Visuals** </br> If available, add visuals for example images to give a better understanding of what is said

## Levels
It is important to know for which of the levels a README is written because the structure can slightly differ and each level addresses different kinds of readers: 

- **Level 1: Linking the main topics** - short description of each main topic and a link to the README file for the specific topic (level 2)
- **Level 2: Main topic with links to specialized topics** -see "Overall Structure"
- **Level 3: Specialized topics with links to technical details** - see "Overall Structure"; can also include more detailed descriptions and examples 
- **Level 4: Technical details and backgrounds** - see "Overall Structure" but can include more technical details and examples

## Dos and Don'ts

**Dos**
- Keep the information up to date
- Add spacing between sections so it is not a block of text
- Use simple and understandable language
- Start with a short overview of the topic 
- Keep a clear structure 

**Don'ts**
- Don't put too much information into the README, keep it relevant
- Avoid personal tone (e.g. you, I)
- It is not a documentation, it is an overview of a project / topic
- Do not overload the README file with too many technical details 
- Avoid writing a block of text without paragraphs / spacing
