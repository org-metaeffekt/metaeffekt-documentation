# Howto Document

The file is intended to provide a structured overview of a topic and should serve as a basis for accessing more detailed files.

# Basic Guide

First, you need to determine the level at which you will write the readme file.

- Level 1: Linking the main topics - fhgjhdfi
- Level 2: Main topic with links to specialized topics
- Level 3: Specialized topics with links to technical details
- Level 4: Technical details and backgrounds

The different levels address different readers, but the readme file should still serve as a simple overview that is 
easy to read and understand for all readers.

## Overall Structure
1. **Headline** </br> Start with a clear and informative title. This should immediately indicate what the document is about. Also possible is to use the folders name as the headline.
2. **Introduction and Description** </br> Introduce the topic and say
   - what it is about (Content)
   - what it is (Purpose)
3. **Linking** </br> All the files in the folder should be linked in the README file. They can either be presented as a bullet point list or included in the text.
4. **Images** </br> If available, images can be included to better visualize the content.
